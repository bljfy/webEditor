# 08 编码与提交流程规范

## 概述
本规范用于统一代码风格、质量门槛与提交流程，确保多人协作下的可读性和可追溯性。

## 功能与职责
- 统一命名、结构、注释和测试要求。
- 约束提交行为，避免“改了但未提交”的状态扩散。
- 为评审与回滚提供稳定基线。

## 编码规范
- 语言与文案：默认中文（简体），配置默认 `meta.language = zh-CN`。
- 命名规范：
  - Python 使用 `snake_case`；
  - JS/TS 变量与函数使用 `camelCase`；
  - 类与组件使用 `PascalCase`。
- 目录职责：业务代码放 `src/`，测试代码放 `test/`，设计与规范放 `docx/`。
- 变更原则：单一职责、最小改动、避免无关重构混入同一次修改。
- 测试要求：功能改动必须补充或更新测试；缺陷修复必须附带回归测试。

## 提交流程（强制）
- 每次修改必须提交一次 commit，不允许仅停留在工作区。
- 一个 commit 只做一类逻辑变更，提交信息应简短、明确、祈使语气。
- 推荐格式：`type(scope): summary`，如 `feat(renderer): add atlas-grid section renderer`。
- 提交前至少完成：自检、必要测试、文档同步（如涉及规范或结构变更）。
